# Liveness Detection по движениям глаз

## Как пользоваться монорепозиторием

1. Работать в своей папке: например `./backend`, `./webview`, `./android`.

2. Для каждой фичи создавать короткоживущую ветку, например `web/svg-animations`.
Когда фича готова, слить `main` в свою ветку (на случай мерж-конфликтов), потом свою ветку в `main`. После этого ветку фичи можно удалить.

## Предполагаемая архитектура

1. Android-приложение загружает WebView

2. WebView запрашивает у сервера стимул (например, `{type: 'moving-dot', x: [], y: [], fps: 60, token: '...'}`) и показывает его.

3. Приложение или WebView записывает видео и отправляет его серверу.

4. Сервер сохраняет видео, преобразует его в траекторию взгляда, вычисляет liveness score и возвращает его клиенту.
